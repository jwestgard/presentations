The Problem

  • Legacy data in spreadsheets;
  • CSV as a de facto standard;
  • CSV = Comma Separated Values;
  • The problem of data ping-pong.

Issues with CSV

  • The comma is a special character (field separator);
  • The line break is a special character (record separator);
  • Character encoding;
  • Not user friendly to edit;
  • Limitations of existing tools;
  • Google Sheets and Open Refine.

CSV File Example

Guiding Principles

  • Excel is here to stay;
  • Solution makes things easier, not more complicated;
  • Data creators should do data cleanup;
  • Solution should be general-purpose.

The Solution

  • CSV-Validate (http://www.github.com/jwestgard/csv-validate/
  • Features
    1. Human-readable CSV schema in the form of a YAML file.
    2. Can be applied to a CSV with a single-line command:
      python3 csv-validate.py -s schema.yaml data.csv
    3. Reads YAML file and the CSV, evaluates the latter against the former, producing a report that identifies violations of the schema by line and column.
    4. It can also work directly from an Excel file.

Sample Output Report

Future Plans

  • Add data cleanup features to facilitate data cleaning;
  • Convert the python script to a GUI application;
  • Bring into alignment with standards (http://www.w3.org/2013/csvw/wiki/Main_Page).
