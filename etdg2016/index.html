SPARQL:
A Query Language for the Semantic Web

Emerging Technologies Discussion Group Presentation

Presented by members of the Libraries Coding Workshop

November 29, 2016

What is SPARQL?

  • SPARQL = "SPARQL Protocol and RDF Query Language"
  • Original 1.0 release by W3C in 2008.
  • Current version (1.1) appeared in 2013.
  • Together with RDF, RDFS, and OWL, SPARQL is a core techology of the semantic web.

What is the Semantic Web?

  • A central feature of the WWW is the use of "hyperlinks" or "jump links" to connect documents.
  • In web 1.0, these links have no meaning, but are merely unidirectional connections between documents.
  • By contrast, in the Semantic Web, there is another kind of linkage, expressed in the form of triples connecting "resources".
  • The links that are created by these triples have a meaning that denotes the nature of the relationship between the two resources that are being connected.

What is the Semantic Web?

  • These links are still fundamentally unidirectional, but they also express an idea or statement about things in the world.
  • The subject, predicate, and object can all be URIs that refer to entities and concepts in the world.
  • For example, a triple might take express the idea that: "LeoTolstoy" "isAuthorOf" "WarAndPeace", but with each of these three concepts expressed as a globally unique identifier, or URI.
  • RDF, the Resource Description Framework, is the way these triples are expressed; there are in turn different syntaxes for expressing RDF (e.g. n-triples, turtle)

Why SPARQL?

  • SPARQL is a query language for RDF.
  • RDF is directed, labeled graph data format:
    • graph: network of vertices and edges (nodes and links);
    • directed: links have a "distinguished direction";
    • labeled: links (predicates) have meaning.
  • RDF is made up of triples, and triples have three components (Subject, Predicate and Object).
  • SPARQL supports aggregation, negation, subqueries, creating values by expressions, and extensible value testing.

SPARQL Syntax

  • A basic query starts with a SELECT clause identifying the variables of interest.
  • Query variables are indicated by the use of either a question mark or dollar sign.
  • The ? or $ are not part of the variable name, so $abc and ?abc are equivalent.
  • The second part of this basic query is a WHERE clause indicating a graph pattern to match against.
  • The graph pattern is in the form subject, predicate, object enclosed by curly braces.
  • The parts of the graph pattern can be expressed as URIs or as a prefix plus an identifier (this shorthand is known as the 'qname' syntax).

SPARQL Functions

  • SPARQL supports various functions that allow the filtering and processing of query results:
    • Functions on strings;
    • Functions on numbers;
    • Functions on date and time;
    • Hash functions.
  • Other "advanced" features include:
    • Pattern matching (regex);
    • Inferencing;
    • Blank nodes.

SPARQL in UMD Libraries

  • SPARQL is likely to be increasingly part of day-to-day work in the Libraries.
  • One of the first ways it is being used is in Fedora 4.
  • Fedora is an implementation of the W3C's Linked Data Platform.
  • Metadata is stored natively as RDF properties on repository containers.
  • Fedora comes bundled with an RDF database or "Triplestore" called Fuseki.
  • One immediate benefit of this infrastructure is that it will become possible to write queries on multiple Fedora-based repositories at once.
  • Another RDF-based system currently being piloted on campus is Vivo, a RDF-based institutional data management framework.

SPARQL in Action

http://nomisma.org/sparql

SPARQL in Action

SELECT * WHERE {
    ?s ?p ?o
    } LIMIT 100

SPARQL in Action

PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
PREFIX nm: <http://nomisma.org/id/>

SELECT ?label WHERE {
    nm:rome skos:prefLabel ?label
    }

SPARQL in Action

SELECT ?label ?type ?field ?lat ?long WHERE {
    nm:rome skos:prefLabel ?label .
    nm:rome rdf:type ?type .
    nm:rome dcterms:isPartOf ?field .
    nm:rome geo:location ?loc .
    ?loc geo:lat ?lat .
    }

SPARQL in Action

SELECT ?label ?type ?field ?lat ?long WHERE {
    nm:rome skos:prefLabel ?label ;
        rdf:type ?type ;
        dcterms:isPartOf ?field ;
        geo:location ?loc .
    ?loc geo:lat ?lat ;
        geo:long ?long
    }

SPARQL in Action

SELECT ?label WHERE {
    nm:rome skos:prefLabel ?label
    } ORDER BY ASC(xsd:string(?label))

SPARQL in Action

SELECT ?label WHERE {
    nm:rome skos:prefLabel ?label .
    FILTER langMatches( lang(?label), "en" )
    }

SPARQL in Action

SELECT ?mints WHERE {
    ?mints a nmo:Mint
    }

SPARQL in Action

SELECT ?label ?lat ?long WHERE {
    ?mints a nmo:Mint ;
        skos:prefLabel ?label ;
        dcterms:isPartOf nm:greek_numismatics ;
        geo:location ?loc .
    ?loc geo:lat ?lat ;
        geo:long ?long
    FILTER langMatches( lang(?label), "en" )
    }
