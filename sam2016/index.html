Dspace

  • More "turn-key" than Fedora
  • Java application, requiring developer time for customization
  • Hosted option (Dspace Direct), and good service provider ecosystem
  • Widely adopted, also internationally
  • Focussed on IR use case
  • Good Google Scholar crawl integration
  • Can be used for image and A/V collections too
  • digital.lib.umd.edu

Current Fedora System

  • Fedora 2-based system, nearly 10 years old
  • Homegrown Java-based admin tools
  • Hippo CMS-based front end
  • Separate DSpace as IR
  • Audio-Video, Photographic, Manuscript, EAD
  • digital.lib.umd.edu

Shortcomings

  • Too much custom code
  • Maintenance headache
  • Processes are too manual and error prone (for example, batch loading)
  • Custom tools are not flexible enough
  • Resources are tight, but the hope is that we will get further as part of the community

Project Goals

  • Leverage repository improvements provided by Fedora 4 application
  • Migrate selected existing services and applications
  • Develop new features
